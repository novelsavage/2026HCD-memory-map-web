### 本プロジェクトでの活用プラン（Phase 2.5 候補）

OSM の弱点（東側が疎・高さが階数からの推定）を PLATEAU LOD1 で置換できる。

1. G空間情報センターから柏市の **GeoJSON（bldg・建物）** をダウンロード
2. `scripts/convert-plateau.mjs`（新規）で bbox 切り出し →
   既存の `{h, pts}` 形式（osm-surroundings.json と同じ）へ変換
3. `surroundings.ts` は**無変更**でそのまま描画できる
4. 帰属表示を「© OpenStreetMap contributors」に加えて
   「出典: 国土交通省 Project PLATEAU 3D都市モデル（柏市）」を追記

- 期待効果: 建物カバー率ほぼ 100%、高さが航空測量の実測値になり
  スカイラインが正確になる。道路・鉄道・駅名は引き続き OSM を使う
- 工数目安: 変換スクリプト半日
- 補足: LOD1 を 3D Tiles でストリーミングする道もあるが、この規模なら
  静的変換で十分軽い（bbox 内なら数 MB 程度の見込み）

## 2. OSM への建物データ貢献

### 作業内容と労力

認識のとおり **3D メッシュは不要**。航空写真の上に建物の外形ポリゴン
（多くは矩形）を描いて `building=yes` タグを付けるだけ。任意で
`building:levels`（階数）を足すと、3D 系レンダラーが高さに変換してくれる
（本サイトの fetch-osm.mjs も levels×3.2m で高さ化している）。

| 項目 | 目安 |
|---|---|
| ツール | ブラウザだけで動く公式 **iD エディタ**（osm.org の「編集」ボタン） |
| 学習コスト | 30 分程度（矩形描画 + `R` キーで直角化 + タグ付け） |
| 1 棟あたり | 20〜40 秒（単純な戸建て矩形） |
| 慣れた人の速度 | 80〜100 棟/時 |
| 東側の疎エリア | 目測 500〜1,500 棟 → **1〜2 人日**（1 人で数セッション） |

### 承認フローの実態: 「事前承認」は存在しない

OSM は Wikipedia 型で、**変更セット（changeset）をアップロードした瞬間に
本番データベースへ反映**される。事前審査・承認者はいない。

- 反映の流れ: 保存 → 即データ反映 → 地図タイルは数分〜数時間で再描画 →
  Overpass API には通常数分で載る（本サイトは fetch-osm.mjs を再実行すれば取り込める）
- 品質管理は**事後**: 地元マッパーが OSMCha 等の監視ツールで新規編集を
  見ていて、問題があれば changeset にコメントが付く（返信するのがマナー）。
  悪質・誤りは revert される。紛争時のみ OSMF の Data Working Group が介入
- つまり「承認待ち」は無いが「見られている」。丁寧な changeset コメント
  （例: "Added missing buildings in Masuo area from Bing imagery"）を書くと良い

### ルール上の重要な注意

1. **Google マップ / Google Earth からのトレースは禁止**（著作権侵害）。
   iD 内で提供される Bing / Esri / Maxar 航空写真 + 現地知識は OK
2. PLATEAU 由来データの OSM 利用は国交省の許諾により日本コミュニティで
   実績があるが、**機械的な大量インポート**の規模になる場合は
   Import Guidelines（talk-ja ML での事前相談 + wiki への手順文書化 +
   インポート専用アカウント）が必要 — ここは事実上の「承認フロー」
3. 手動で数百棟をトレースする分には通常編集の範囲で、相談不要

### 推奨の進め方

- **サイトの見た目を良くするのが目的なら Phase 2.5（PLATEAU 変換）が最速・最正確**。
  OSM 貢献は反映がタイル/Overpass 経由で間接的
- OSM 貢献は「地域への恩返し + 麗澤大の建物に name タグを付けて
  地図上の大学を充実させる」価値がある。まず iD で 1〜2 時間、
  大学周辺を触ってみるのがおすすめ（アカウントは無料）
- 本気で東側を埋めるなら、手動トレースを数人で分担するか、
  PLATEAU インポートを talk-ja に相談する 2 択
